# tenx-brand

Single source of truth for TenX Protocols brand assets used in generated
internal documents — colors, logo, typography, and the Pandoc
reference-document template that gives every synced policy and template a
consistent title page / header bar, without editing each document
individually.

This is the **parent company's** brand. The Delaware spinout (AI Workflow
Platform) will get its own brand identity and its own equivalent repo once
that's defined — this repo isn't meant to be reused for that entity.

## Contents

- [colors.md](colors.md) — color palette (confirmed + derived)
- [typography.md](typography.md) — typeface decision for generated documents
- `logo/` — see below
- `pandoc/build-reference.py` — regenerates all four reference docs below
- `pandoc/generate-cover-art.py` — regenerates `pandoc/assets/report-cover-art.png`,
  the report cover's decorative graphic

## Document tracks

Four Pandoc `--reference-doc` templates, one per document track, chosen
per document by its detected doc type. All four share the same fonts,
heading colors, and footer machinery (`base_document()` in
`build-reference.py`) — only the title-page treatment differs:

| Track | Reference doc | Look |
|---|---|---|
| Policy | `reference.docx` | Restrained and centered: a large logo anchors the cover, the title sits at the page's true vertical center below it, plain violet text, no color panel. For compliance policy documents specifically — SOC 2 / ISO 27001 and similar converge on exactly this kind of typographic restraint. |
| Document | `reference-document.docx` | Left-aligned with a thin accent rule beside the title (a "quote bar" headline treatment) and a small corner logo. For living operational/legal records — runbooks, the legal index — that need a real title page but aren't compliance policy or a periodic report. |
| Report | `reference-report.docx` | A full-bleed graphic cover: a violet masthead band (small logo + decorative art) above a violet title panel, both bleeding most of the way to the page edge. For periodic reports (quarterly business reports and similar) — the "annual report / pitch deck" genre the other three tracks deliberately avoid. |
| Template | `reference-template.docx` | No title page at all — blank fillable forms never carry YAML frontmatter, so Pandoc's title-block (and this doc's Title/Author/Date styles) never fire. Just the small logo and normal page numbers on every page, including page 1. |

The report cover's decorative graphic (`pandoc/assets/report-cover-art.png`,
generated by `generate-cover-art.py`) reuses the icon mark itself — two
whole, quiet, tone-on-tone copies of it — rather than a generic stock
wireframe/hexagon-network graphic or the icon's individual triangles
scattered apart (an earlier attempt at the latter read as disconnected
geometric confetti, not a recognizable brand extension).

## Logo

`logo/build-logos.py` regenerates every variant from `logo/source/icon.svg`
and `logo/source/wordmark.svg` — the original brand mark's path data,
extracted once from the source SVG pulled from tenx.inc. Edit those two
files (not the script) if the mark itself ever changes.

Four compositions, each in black and white:

| Variant | What it is |
|---|---|
| `icon` | The geometric mark alone |
| `wordmark` | "TenX" alone |
| `lockup-horizontal` | Icon beside the wordmark (the original combined mark) |
| `lockup-stacked` | Icon centered above the wordmark |

Each composition produces one padded SVG in `logo/svg/` (vector, the real
source of truth for that variant) and PNG rasters at 128/256/512/1024px
tall in `logo/png/`, generated from the SVG via `rsvg-convert` (not
ImageMagick's own SVG delegate — see the script's `rasterize()` docstring
for why: ImageMagick's delegate shells out to `rsvg-convert` internally
and was found to crash specifically on the stacked lockup's `<g
transform>` group; direct invocation renders the identical file with no
issue). Every SVG/PNG has transparent background and padding around the
mark so it never touches the image edge.

## How other repos use this

`tenx-doc-templates` and `tenx-documents`' sync pipelines check out this
repo in CI and pass one of the four `pandoc/*.docx` files to Pandoc via
`--reference-doc`, selected per document by its detected doc type. Update
a color, the logo, or the reference-doc build script here once; every
document picks it up on its next sync — no per-document styling.
